# yt-dlp must be kept current — YouTube rotates their player API roughly
# every 2–3 months and old versions hit "SSL: UNEXPECTED_EOF" / "Sign in
# to confirm you're not a bot". The Dockerfile runs an explicit
# `pip install --upgrade yt-dlp` step *after* this requirements pass,
# which always pulls the latest stable release — so this floor only has
# to be loose enough to install at all. Don't pin a specific YYYY.MM.DD:
# yt-dlp's nightly builds are `.dev0` and pip refuses to install dev
# releases by default, which means a floor like `>=2026.4.0` becomes
# unsatisfiable until the matching stable lands (HF build = BUILD_ERROR).
# A loose `>=YYYY.M` floor is safe — month-level pinning matches yt-dlp's
# actual stable release cadence.
yt-dlp>=2026.3
